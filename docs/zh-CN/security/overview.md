# 安全概览

SecuClaw的设计以安全为核心。本文档概述了安全功能和最佳实践。

## 安全架构

### 纵深防御

SecuClaw实施多层安全：

```
┌─────────────────────────────────────────────────────┐
│                   网络层                              │
├─────────────────────────────────────────────────────┤
│                   应用层                              │
├─────────────────────────────────────────────────────┤
│                   数据层                              │
├─────────────────────────────────────────────────────┤
│                   身份层                              │
└─────────────────────────────────────────────────────┘
```

### 关键安全功能

#### 1. 认证

- **基于令牌的认证**: 安全的WebSocket连接
- **OAuth支持**: 用于LLM提供商认证
- **API密钥管理**: 加密的凭证存储

#### 2. 授权

- **基于角色的访问控制(RBAC)**: 细粒度权限
- **会话隔离**: 每个代理、用户、渠道隔离
- **渠道权限**: 按消息渠道控制访问

#### 3. 数据保护

- **静态加密**: 敏感数据加密
- **TLS传输**: 所有通信加密
- **安全存储**: 凭证安全存储

#### 4. 沙盒化

- **基于Docker的隔离**: 工具在容器中运行
- **资源限制**: CPU、内存、网络限制
- **网络策略**: 出站流量控制

## 安全最佳实践

### 部署

1. **在生产环境中使用HTTPS**
   ```json5
   {
     gateway: {
       tls: {
         enabled: true,
         cert: "/path/to/cert.pem",
         key: "/path/to/key.pem",
       },
     },
   }
   ```

2. **限制网络访问**
   - 使用防火墙规则
   - 在专用网络中部署
   - 使用VPN进行远程访问

3. **启用认证**
   ```json5
   {
     gateway: {
       auth: {
         token: "${GATEWAY_TOKEN}",
       },
     },
   }
   ```

### 配置

1. **使用白名单**
   ```json5
   {
     channels: {
       api: {
         allowFrom: ["10.0.0.0/8", "192.168.0.0/16"],
       },
     },
   }
   ```

2. **为工具启用沙盒化**
   ```json5
   {
     agents: {
       defaults: {
         sandbox: {
           mode: "all",
           scope: "session",
         },
       },
     },
   }
   ```

3. **定期验证配置**
   ```bash
   secuclaw doctor
   ```

### 运维

1. **监控安全日志**
   ```bash
   secuclaw logs --level security
   ```

2. **定期更新**
   ```bash
   secuclaw update
   ```

3. **备份配置**
   ```bash
   cp ~/.secuclaw/secuclaw.json ~/.secuclaw/backup/
   ```

## 合规性

SecuClaw支持以下合规性：

- **SOC 2**: 安全、可用性、保密性
- **ISO 27001**: 信息安全管理
- **GDPR**: 数据保护
- **SCF 2025**: 安全控制框架

## 事件响应

如果您怀疑发生安全事件：

1. **隔离**: 禁用受影响的渠道
2. **调查**: 查看日志
   ```bash
   secuclaw logs --since "1 hour ago"
   ```
3. **修复**: 应用修复
4. **报告**: 记录发现
