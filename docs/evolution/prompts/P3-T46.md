## 输出要求

### 必须产出的文件

1. **trait.ts** - Provider Trait定义
   ```typescript
   // Provider能力接口
   export interface IProviderCapabilities {
     readonly streaming: boolean;
     readonly tools: boolean;
     readonly vision: boolean;
     readonly audio: boolean;
     readonly maxContextLength: number;
     readonly supportedModels: string[];
   }
   
   // Provider主接口
   export interface ILLMProvider {
     readonly id: string;
     readonly name: string;
     readonly capabilities: IProviderCapabilities;
     
     // 核心方法
     chat(messages: ChatMessage[], options?: ChatOptions): Promise<ChatResponse>;
     chatStream(messages: ChatMessage[], options?: ChatOptions): AsyncIterable<ChatResponse>;
     embed(text: string): Promise<number[]>;
     countTokens(text: string): number;
     
     // 生命周期
     initialize?(): Promise<void>;
     dispose?(): Promise<void>;
     
     // 健康检查
     healthCheck(): Promise<boolean>;
   }
   
   // Provider配置接口
   export interface IProviderConfig {
     apiKey?: string | Secret<string>;
     baseUrl?: string;
     model?: string;
     timeout?: number;
     retries?: number;
   }
   
   // Provider工厂接口
   export interface IProviderFactory {
     create(config: IProviderConfig): ILLMProvider;
     validate(config: IProviderConfig): boolean;
   }
   ```

2. **base.ts** - 基类实现（重构）
   - `BaseLLMProvider` 抽象类（重构）
   - `ProviderCapabilities` 实现
   - 通用错误处理
   - 通用重试逻辑
   - 通用超时处理

3. **config.ts** - 配置类型
   - `ProviderConfigBuilder` 类
   - `validateConfig()` - 配置验证
   - `mergeConfig()` - 配置合并
   - `resolveSecret()` - 解析Secret

4. **registry.ts** - 提供商注册表（重构）
   - `ProviderRegistry` 类
   - `register()` - 注册提供商
   - `get()` - 获取提供商
   - `list()` - 列出所有提供商
   - `getById()` - 按ID获取
   - `getByCapability()` - 按能力获取

5. **factory.ts** - 工厂函数（重构）
   - `createProvider()` - 创建提供商
   - `createProviderFromConfig()` - 从配置创建
   - `autoDetectProvider()` - 自动检测

6. **测试文件**
   - trait.test.ts
   - base.test.ts
   - registry.test.ts
   - factory.test.ts
