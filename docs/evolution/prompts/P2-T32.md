## 任务描述

实现Zeroizing<String>类型，确保敏感数据（API密钥、密码等）在不再需要时自动从内存中安全擦除，防止内存转储攻击。

## 输出要求

### 必须产出的文件

1. **secret.ts** - Secret<T>包装
   ```typescript
   export class Secret<T extends Zeroizable> {
     private value: T;
     private wiped: boolean;
     
     // 创建Secret
     static from<T>(value: T): Secret<T>;
     
     // 访问值（回调方式）
     with<R>(fn: (value: T) => R): R;
     
     // 手动擦除
     wipe(): void;
     
     // 检查状态
     isWiped(): boolean;
     
     // 克隆（创建副本）
     clone(): Secret<T>;
   }
   ```

2. **zeroize.ts** - 擦除实现
   - `Zeroizable` 接口
   - `zeroize()` - 擦除函数
   - `zeroizeBuffer()` - 擦除Buffer
   - `zeroizeString()` - 擦除字符串
   - `zeroizeArray()` - 擦除数组
   - `Zeroizing<T>` 装饰器

3. **buffer.ts** - 安全Buffer
   - `SecureBuffer` 类
   - `alloc()` - 分配安全内存
   - `from()` - 从数据创建
   - `wipe()` - 擦除
   - `toString()` - 转字符串（返回副本）
   - `equals()` - 比较

4. **memory.ts** - 内存操作
   - `secureCompare()` - 常量时间比较
   - `secureRandom()` - 安全随机数
   - `wipeMemory()` - 擦除内存区域
   - `lockMemory()` - 锁定内存（防止交换）

5. **types.ts** - 类型定义
   - `Zeroizable` - 可擦除接口
   - `Secret<T>` - 秘密类型
   - `SecureBuffer` - 安全Buffer类型
   - `WipeCallback` - 擦除回调

6. **测试文件**
   - secret.test.ts
   - zeroize.test.ts
   - buffer.test.ts
   - memory.test.ts

## 技术约束

### 必须遵守
- 使用FinalizationRegistry实现自动清理
- 使用Buffer进行安全擦除
- 使用常量时间比较
- 支持泛型类型

### 禁止
- 不要在日志中输出Secret值
- 不要使用普通字符串存储敏感数据
- 不要跳过擦除步骤
