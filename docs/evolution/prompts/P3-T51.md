## 输出要求

### 必须产出的文件

1. **runtime.ts** - WASM运行时
   ```typescript
   export class WasmRuntime {
     private instance: WebAssembly.Instance;
     private metering: FuelMetering;
     private epoch: EpochInterrupt;
     
     // 加载WASM模块
     static async load(module: BufferSource, config: WasmConfig): Promise<WasmRuntime>;
     
     // 调用函数
     call<T>(fn: string, args: unknown[]): Promise<T>;
     
     // 获取内存
     getMemory(): WebAssembly.Memory;
     
     // 销毁
     dispose(): void;
   }
   ```

2. **metering.ts** - 燃料计量
   - `FuelMetering` 类
   - `setFuel()` - 设置燃料
   - `getFuel()` - 获取剩余燃料
   - `consumeFuel()` - 消耗燃料
   - `syncFuel()` - 同步燃料状态
   - 支持动态调整

3. **epoch.ts** - 时代中断
   - `EpochInterrupt` 类
   - `startEpoch()` - 开始时代
   - `endEpoch()` - 结束时代
   - `checkInterrupt()` - 检查中断
   - `setDeadline()` - 设置截止时间
   - 看门狗线程支持

4. **host.ts** - 主机函数
   - `HostFunctions` 类
   - `registerFunction()` - 注册主机函数
   - `getImports()` - 获取导入对象
   - 预定义函数：
     - `log()` - 日志输出
     - `clock()` - 时钟
     - `random()` - 随机数
     - `store()` - 存储
     - `fetch()` - 网络请求（受限）

5. **limits.ts** - 资源限制
   - `ResourceLimits` 类
   - `setMemoryLimit()` - 设置内存限制
   - `setCpuLimit()` - 设置CPU限制
   - `setTimeLimit()` - 设置时间限制
   - `setNetworkLimit()` - 设置网络限制
   - `checkLimits()` - 检查限制

6. **watchdog.ts** - 看门狗
   - `Watchdog` 类
   - `start()` - 启动看门狗
   - `stop()` - 停止看门狗
   - `feed()` - 喂狗
   - `isAlive()` - 检查存活
   - 超时自动终止

7. **policy.ts** - 沙箱策略
   - `SandboxPolicy` 类
   - `allowFs()` - 允许文件系统
   - `allowNet()` - 允许网络
   - `allowEnv()` - 允许环境变量
   - `allowProcess()` - 允许子进程
   - 预定义策略模板

8. **executor.ts** - 沙箱执行器
   - `SandboxExecutor` 类
   - `execute()` - 执行代码
   - `executeScript()` - 执行脚本
   - `executeBinary()` - 执行二进制
   - 返回执行结果

9. **types.ts** - 类型定义
   - `WasmConfig` - WASM配置
   - `ResourceLimits` - 资源限制
   - `ExecutionResult` - 执行结果
   - `SandboxPolicy` - 沙箱策略

10. **测试文件**
    - runtime.test.ts
    - metering.test.ts
    - epoch.test.ts
    - executor.test.ts
