## 输出要求

### 必须产出的文件

1. **recovery.ts** - 恢复机制
   ```typescript
   export class SessionRecovery {
     // 7阶段恢复
     async recover(sessionId: string): Promise<RecoveryResult>;
     
     // 阶段1: 检测损坏
     private detectCorruption(session: Session): CorruptionReport;
     
     // 阶段2: 验证消息序列
     private validateSequence(messages: Message[]): ValidationResult;
     
     // 阶段3: 识别断点
     private identifyBreakpoint(validation: ValidationResult): number;
     
     // 阶段4: 加载检查点
     private loadCheckpoint(sessionId: string, seq: number): Checkpoint | null;
     
     // 阶段5: 重建状态
     private rebuildState(checkpoint: Checkpoint, messages: Message[]): SessionState;
     
     // 阶段6: 验证恢复
     private verifyRecovery(state: SessionState): boolean;
     
     // 阶段7: 持久化
     private persist(session: Session): Promise<void>;
   }
   ```

2. **validator.ts** - 消息验证
   - `MessageValidator` 类
   - `validateMessage()` - 验证单条消息
   - `validateSequence()` - 验证消息序列
   - `checkIntegrity()` - 检查完整性
   - `detectGaps()` - 检测缺口
   - `detectDuplicates()` - 检测重复

3. **repair.ts** - 修复逻辑
   - `SessionRepair` 类
   - `repairSequence()` - 修复序列
   - `fillGaps()` - 填补缺口
   - `removeDuplicates()` - 移除重复
   - `reconstructContext()` - 重建上下文

4. **checkpoint.ts** - 检查点
   - `CheckpointManager` 类
   - `createCheckpoint()` - 创建检查点
   - `loadCheckpoint()` - 加载检查点
   - `listCheckpoints()` - 列出检查点
   - `pruneCheckpoints()` - 清理旧检查点
   - `getLatestCheckpoint()` - 获取最新检查点

5. **types.ts** - 类型定义扩展
   - `RecoveryResult` - 恢复结果
   - `CorruptionReport` - 损坏报告
   - `ValidationResult` - 验证结果
   - `Checkpoint` - 检查点类型
   - `RecoveryOptions` - 恢复选项

6. **测试文件**
   - recovery.test.ts
   - validator.test.ts
   - repair.test.ts
   - checkpoint.test.ts
